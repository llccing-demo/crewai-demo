## 选题 1：现代服务端渲染架构碰撞：Angular Zoneless/SSR 对决 React Server Components (RSC) 
**定位**：通过横向对比 Angular 现代架构与 React RSC，帮助开发者在 2026 年的前端技术选型中建立全局视野。 
**写作要点**： 
- 回顾 Angular 19+ 中由 Zone.js 转向 Signals 并深度融合 SSR、Hydration 与 BFF 的架构演进。 
- 填补技术认知空白：深入剖析 2026 年已成行业标准的 React Server Components (RSC) 的心智模型与零打包体积优势。 
- 核心对决：探讨两种范式在服务端状态管理、流式传输（Streaming）机制以及路由层面的异同点与适用业务场景。 
**目标读者**：面临技术选型的高级前端工程师、全栈开发者，以及想要对比 Angular 与 React 现代底层架构差异的技术 Leader。

## 选题 2：补齐现代工程化最后拼图：在 Turborepo + GitHub Actions 中落地 Vitest 与 Playwright 
**定位**：结合已有的 Monorepo 和 CI/CD 实践，展示如何构建下一代极速、自动化的前端全链路测试基建。 
**写作要点**： 
- 趋势解读：分析为何在 2026 年 Vitest 与 Playwright 已经彻底取代 Jest 与 Cypress，成为现代前端工程化的新标准。 
- 架构整合实战：详细记录在 pnpm workspace + Turborepo 架构中优雅配置 Vitest 单元测试边界的最佳实践。 
- CI/CD 进阶：将 Playwright 端到端测试无缝集成至现有的 GitHub Actions 工作流，实现前端发布的自动化质量防御。 
**目标读者**：关注代码质量的前端开发者、测试工程师 (QA)，以及负责搭建团队 CI/CD 基础架构的 DevOps 工程师。

## 选题 3：打造保护隐私的极客开发流：基于 OpenClaw 构建本地私有化 AI 编程助手 
**定位**：将宏观的 AI 探索落地到日常编码场景，提供一套无需担心数据泄露的本地大模型辅助编程最佳实践。 
**写作要点**： 
- 结合之前的 OpenClaw 自托管实战经验，分析本地化 AI 大模型在日常代码补全与重构中的独特优势（隐私保护与低延迟）。 
- 破解开发痛点：探讨如何让本地部署的大模型真正理解并索引整个 Monorepo（大型代码库）的全局上下文。 
- 生产力实战：演示将私有大模型与主流现代 IDE（如 Cursor / VS Code）深度融合的具体配置与 Prompt 调优技巧。 
**目标读者**：关注代码隐私与企业数据安全的极客开发者，以及对本地大模型（Local LLM）部署与 AI 提升生产力感兴趣的程序员。

## 选题 4：前端基建大换血：从 Webpack 迁移到 Rspack 带来的极速构建体验与底层解析 
**定位**：填补博客在下一代基于 Rust 的构建工具领域的空白，探讨底层编译性能飞跃对大型前端项目的影响。 
**写作要点**： 
- 盘点 2026 年前端构建工具链的底层演进：Webpack 的市场衰退与以 Rspack/Vite 为代表的 Rust/Go 工具链的全面崛起。 
- 迁移实战复盘：记录将一个包含复杂依赖（或之前的 Angular/Vue 旧项目）迁移至 Rspack 的踩坑指南与最终收益分析。 
- 结合 Turborepo 放大优势：展示底层构建速度的指数级提升，是如何与 Turborepo 的缓存机制结合，成倍放大 Monorepo 协同开发效率的。 
**目标读者**：受困于大型项目构建速度瓶颈的前端开发者、关注性能优化的前端架构师，以及对底层编译工具链演进感兴趣的技术极客。

## 选题 5：当 AI Agent 遇见 BFF 架构：基于 OAuth 2.1 构建安全可控的智能前端应用 
**定位**：将博主最擅长的安全认证（OAuth 2.1）、BFF 架构与最前沿的 AI 智能体应用进行硬核的跨界融合。 
**写作要点**： 
- 风险洞察：探讨 2026 年前端直连调用 AI Agent 服务（如之前的 Stripe 集成）时面临的跨域、鉴权漏洞与 API Key 泄露风险。 
- 架构设计创新：利用 BFF（Backend For Frontend）作为 AI Agent 的代理通信层，安全地隐藏真实调用逻辑与计费大模型接口。 
- 安全底座加固：结合 OAuth 2.1 的 PKCE 机制，为前端与 BFF 之间的 AI 智能体会话交互建立坚不可摧的认证防线。 
**目标读者**：正在探索 AI 产品商业化落地的全栈开发者、关注 Web 安全架构的高级工程师，以及 SaaS 平台的后端与前端架构师。
